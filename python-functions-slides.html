Overview

Python functions are essential building blocks for organizing code, improving reuse, and ensuring modularity. This presentation covers:

  • Types of functions

  • Returning values
  • Execution flow

What is a Function?

A function is a named sequence of statements that performs a specific task and can be called by its name from other parts of the code.

Functions help:

  • Reduce code repetition
  • Improve readability
  • Organize complex programs
  • Enable better testing and debugging

Types of Functions

Python supports three main types of functions:

Type Description Example
Built-in Provided by Python interpreter print(), len()
Module-defined Available through imported modules math.sqrt(), random.randint()
User-defined Created by programmers def greet(): ...

Built-in Functions

Python comes with numerous built-in functions always available without importing:

  • print() - Display output
  • len() - Get length of object
  • abs() - Absolute value
  • input() - Get user input
  • max(), min(), sum() - Aggregate functions
Example:
print(abs(-20)) # Output: 20 result = sum([1, 2, 3, 4]) # Output: 10

Module-defined Functions

Modules (external libraries) provide specialized functions that extend Python's functionality.

Common modules: math, os, sys, random, datetime

Example using math module:
import math area = math.pi * math.pow(3, 2) # Circle area print(area) # Output: 28.274...

User-defined Functions

Programmers can define their own functions to fit specific needs.

Basic Syntax:

def function_name(parameters): """Docstring""" statements return value
Simple Example:
def greet(name): print(f"Hello, {name}")

Creating User-defined Functions

Step-by-step process:

  1. Use the def keyword
  2. Name the function (follow naming conventions)
  3. Specify parameters (optional)
  4. Add the function body with proper indentation
  5. Use return for output (optional)
Complete Example:
def square(x): """Returns the square of a number""" return x * x result = square(5) # result = 25

Arguments and Parameters

Parameter: A variable in the function definition.

Argument: The value supplied during function invocation.

Clarification with Example:
def multiply(a, b): # a, b are parameters return a * b result = multiply(3, 7) # 3, 7 are arguments print(result) # Output: 21

Note: Parameters are defined in the function signature, while arguments are passed at the call site.

Default Parameters

Default parameters allow functions to use predefined values when arguments are not provided.

Example:
def greet(name, message="Hello"): print(f"{message}, {name}") greet("Priya") # Output: Hello, Priya greet("Priya", "Hi") # Output: Hi, Priya

Important: Default parameters must come after non-default parameters.

Positional Parameters

With positional parameters, the order of arguments matters.

Example:
def display(name, age): print(f"{name} is {age} years old.") display("Rahul", 25) # Correct display(25, "Rahul") # Wrong - produces error

Tip: Mix with keyword arguments for clarity:

display(age=25, name="Rahul") # Order doesn't matter

Function Returning Value(s)

Functions can return single or multiple values using the return statement.

Single value:

def add(a, b): return a + b

Multiple values (as tuple):

def calc(x, y): return x + y, x - y, x * y sum_val, diff_val, prod_val = calc(5, 3)

Flow of Execution

The flow of execution determines how code runs sequentially in a program.

Process:

  • Main program starts execution
  • Function call transfers control to the function
  • Executes all statements in the function body
  • Returns to the calling code upon return statement or function end
  • Main program continues from the next line after function call
Main Program ↓ Function Call ↓ Function Body (Execution) ↓ Return to Caller

Scope of a Variable

Variable scope determines where a variable is accessible in the code.

Scope Description Accessibility
Local Defined within a function Only within that function
Global Defined outside all functions Accessible throughout the program

Local Scope Example

Variables defined inside a function are local and inaccessible outside:

Code:
def foo(): message = "Hello" print(message) # Accessible here foo() # Output: Hello print(message) # ERROR: NameError - not defined outside

Key Point: Local variables are created when the function is called and destroyed when it exits.

Global Scope Example

Variables declared outside functions are global and accessible anywhere:

Code:
count = 10 def show(): print(count) # OK - can access global show() # Output: 10 print(count) # OK - accessible in main

Important: Global variables exist throughout the program's execution.

Modifying Global Variables

To modify global variables inside a function, use the global keyword:

Without `global` keyword:
total = 0 def increment(): total += 1 # ERROR: UnboundLocalError increment()
With `global` keyword:
total = 0 def increment(): global total total += 1 increment() print(total) # Output: 1

Best Practices

  • Use meaningful names: Function names should clearly describe what they do
  • Keep functions small: Each function should do one thing well
  • Document with docstrings: Explain purpose, parameters, and return values
  • Avoid global variables: Minimize use to prevent side effects
  • Test independently: Test functions in isolation
  • Use default parameters: Provide sensible defaults when appropriate
